**文档状态**：需求初稿

---

## 一、项目概述

### 1.1 项目背景
Rewind.ai 是一个创新型梦境记录与社交平台，提供普通梦境记录、沉浸式白日梦体验、社交互动等功能。为了保障平台的健康运营、内容安全和用户管理，需要建设一套功能完善的后台管理系统。

### 1.2 建设目标
- 实现用户全生命周期管理
- 保障平台内容安全与质量
- 提供数据统计与运营决策支持
- 实现高效的平台运维管理
- 提供完善的系统配置能力

---

## 二、系统用户角色

| 角色名称 | 角色描述 | 主要权限 |
|---------|---------|---------|
| 超级管理员 | 系统最高权限管理者 | 全部功能权限 |
| 运营管理员 | 负责平台日常运营 | 内容审核、用户管理、数据统计 |
| 内容审核员 | 专门负责内容审核 | 梦境审核、评论审核、用户举报处理 |
| 客服专员 | 处理用户反馈与投诉 | 用户查询、工单处理、权限受限 |
| 财务管理员 | 管理钱包与交易 | 钱包管理、交易记录、奖励配置 |
| 系统运维员 | 负责系统技术维护 | 系统配置、日志查看、监控告警 |

## 三、功能模块详细设计

### 模块一：首页仪表盘

#### 1.1 数据概览卡片
| 指标名称 | 统计逻辑 | 展示方式 |
|---------|---------|---------|
| 总用户数 | 累计注册用户数 | 数字卡片 + 日/周环比 |
| 活跃用户数 | 今日/本周活跃用户 | 数字卡片 + 趋势 |
| 梦境总数 | 累计发布梦境数 | 数字卡片 + 日增量 |
| 今日新增梦境 | 今日新增梦境数 | 数字卡片 |
| 互动总数 | 点赞+评论+打赏总数 | 数字卡片 |
| 待审核内容 | 待审核梦境+评论数 | 数字卡片 + 红色警示 |

- **用户增长趋势**：按日/周/月展示新增用户曲线图
- **梦境发布趋势**：按日/周/月展示梦境发布量
- **互动活跃趋势**：点赞、评论、打赏的分时趋势图
- **热门内容排行**：点赞/浏览/评论最多的梦境TOP10

#### 1.3 快捷入口
- 待审核内容快捷入口
- 最新用户注册列表
- 异常行为告警

### 模块二：用户管理系统

#### 2.1 用户列表管理
| 功能点 | 功能描述 |
|-------|---------|
| 用户查询 | 支持按用户ID、昵称、手机号、邮箱模糊搜索 |
| 筛选条件 | 按注册时间、用户状态、性别、活跃状态筛选 |
| 列表展示 | 用户头像、昵称、用户ID、注册时间、最后登录、梦境数、粉丝数、关注数、状态 |
| 批量操作 | 批量封禁、批量解禁、批量导出 |
| 排序 | 按注册时间、最后登录时间、梦境数、粉丝数排序 |

#### 2.2 用户详情页
**基本信息**：
- 用户头像、昵称、用户名、手机号、邮箱
- 性别、生日、个人简介
- 注册时间、最后登录时间、登录IP
- 设备信息、注册渠道

**社交数据**：
- 关注数、粉丝数、互相关注数
- 发布梦境数、公开梦境数、私密梦境数
- 获赞数、收到评论数、收到打赏数

**操作记录**：
- 登录日志（时间、IP、设备）
- 操作日志（发布、修改、删除等关键操作）
- 奖惩记录（封禁记录、奖励记录）

**钱包信息**：
- 当前梦想币余额
- 累计收入、累计支出
- 最近交易记录

**管理操作**：
- 编辑用户信息
- 重置用户密码
- 封禁/解禁用户（支持设置封禁时长、填写封禁原因）
- 强制用户退出登录
- 查看用户发布的所有梦境
- 查看用户的关注/粉丝列表
- 导出用户数据

#### 2.3 用户分层管理
- **普通用户**：默认用户组
- **优质创作者**：标记优质内容创作者，给予推荐权重
- **官方账号**：平台官方账号标识
- **黑名单用户**：违规用户，限制部分功能

#### 2.4 用户认证管理
- 实名认证状态查看与审核
- 身份信息管理

## 四、非功能需求

### 4.1 性能需求
- 页面响应时间 < 2秒
- 列表查询时间 < 1秒
- 支持并发用户数 ≥ 100
- 支持数据量：用户千万级、内容亿级

### 4.2 安全需求
- 所有操作需要身份认证
- 敏感操作需要二次验证
- 权限控制精确到按钮级别
- 密码加密存储
- 防SQL注入、XSS攻击
- 完整的操作审计日志

### 4.3 可用性需求
- 系统可用性 ≥ 99.9%
- 数据定时备份
- 支持灰度发布
- 异常快速恢复

### 4.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 响应式设计，支持PC端和Pad端
- 支持1080p及以上分辨率

## 五、技术架构建议

### 5.1 前端技术栈
- **框架**：Vue 3 + TypeScript
- **UI组件库**：Element Plus / Ant Design Vue
- **状态管理**：Pinia
- **路由**：Vue Router
- **图表库**：ECharts
- **HTTP客户端**：Axios

### 5.2 后端技术栈
- **框架**：Spring Boot 3.x（复用现有架构）
- **安全**：Spring Security + JWT
- **ORM**：Spring Data JPA
- **数据库**：MySQL 8.0
- **缓存**：Redis
- **搜索引擎**：Elasticsearch（用于内容搜索）
- **消息队列**：RabbitMQ（异步处理、日志审计）

### 5.3 部署架构
- 前后端分离部署
- Nginx反向代理
- Docker容器化
- 支持集群部署
